## Workarounds applied
1. **`.git/*.lock` immutable in sandbox** ‚Äî same recurring blocker. `rm -f` returns EPERM but `mv` is allowed, so I renamed the stale locks out of the way (`.lock.z1`, `.lock.z2`, etc.) before each git op. Had to redo this before `git commit` and again before `git push`. Git itself then succeeds despite the lingering warnings.
2. **`.git/refs/heads/main.lock.dead2` from prior cleanup** ‚Äî my first pre-flight pass renamed `main.lock` ‚Üí `main.lock.dead2` inside `refs/heads/`, which git then tried to parse as a ref called `main.lock.dead2` and choked with `fatal: bad object refs/heads/main.lock.dead2`. Moved it outside `refs/` to `.git/trash-main-lock-dead2` and the log/fetch/commit path recovered.
3. **`dist/xsantcastx/3rdpartylicenses.txt` EPERM on unlink** ‚Äî same FUSE permission issue that hit prior runs. Worked around by `mv dist dist.stale-<epoch>` before the production build so ng could create a fresh `dist/`.
